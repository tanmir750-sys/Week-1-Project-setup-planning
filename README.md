# Week 1: Project setup & planning
Objectives
Set up Python project structure and virtual environment
Initialize Git repository with .gitignore
Write a Game Design Document (GDD) — rules, word categories, scoring
Brainstorm 50+ emoji-to-word puzzles manually
Install pygame/tkinter early and verify it opens a blank window
Task Breakdown
Project structure & virtual environment — Create folders, set up a venv, add a
requirements.txt, and confirm Python runs inside it.
Git repository — Run git init, add a .gitignore, make an initial commit, and push to GitHub.
Design doc — Define wrong-guess limit, categories, difficulty tiers, and a scoring outline in
DESIGN.md.
Emoji-to-word mapping — Decide one-emoji vs emoji-set per word, test a few samples, and
document the format.
Word list — Brainstorm 50+ words, tag each with difficulty, and assign an emoji to each.
Deliverables
Game Design Document
project repository
50 emoji puzzles
pygame/tkinter installed
Tools & Resources
Python 3.11+
venv
Git.
GitHub
VS Code/pycharm
Markdown
